# エンジン引き継ぎパック(バランス側 → エンジン実装)

Date: 2026-07-05
対象読者: ゲームエンジン実装者
前提: RULE_SPEC.md と terminology.md を読んでいること

このドキュメントは、バランス調整側で確定済みの (1) ゲーム定数、(2) RULE_SPEC本文に未反映の裁定、(3) 動作確認用の素体カードセット をまとめたもの。**RULE_SPECと本パックの2つで、仕様の食い違いなく実装できる**ことを意図している。

## 1. ゲーム定数(constants v1)

**実装上のお願い: 定数はハードコードせず、設定ファイル(JSON等)から読み込む作りにしてください。** バランス改訂は今後も続くため、シミュレーションとエンジンが同じ設定ファイルを共有できる形が理想です。

```json
{
  "BOARD_SIZE": 3,
  "WIN_CONTROL": 5,
  "START_LIFE": 15,
  "START_HAND": 5,
  "MAX_HAND": 5,
  "DECK_SIZE": 24,
  "MANA_PER_TURN": 3,
  "MAX_MANA": 15,
  "START_MANA_FIRST": 3,
  "START_MANA_SECOND": 4,
  "DESTROY_MANA_GAIN": 1,
  "BLIND_SPOT_BONUS": 2,
  "ATTRIBUTE_HP_BONUS": 2,
  "MAX_HP": 10
}
```

シミュレーション約10万試合で検証済みの値。根拠は `docs/balance-overview.md` を参照。

## 2. RULE_SPEC本文に未反映の裁定(重要)

RULE_SPEC.md の改訂が追いつくまで、以下4件は**本文の記載より優先**して実装すること。

### 裁定1: 再命令は「回転 / 回転攻撃 / 攻撃」の3択(2026-07-07改定)

RULE_SPEC §9.3 の記載に関わらず、再命令は以下の3択で実装する:

- **回転**: 90度回転。回転枠を消費
- **回転攻撃**: 攻撃の直前に90度回転を1回行ってよい。コストは再命令1回分のみ。**回転枠と攻撃枠の両方を消費**
- **攻撃**: 従来どおり。攻撃枠を消費

各式神のターン上限は「回転1回まで・攻撃1回まで」。**攻撃の後に向きを変える手段は存在しない**。回転攻撃の攻撃範囲・弱点判定は回転後の向きで行う。召喚時攻撃には回転は付かない(召喚時は向きを自由に選べるため)。

### 裁定2: 撃破された式神は反撃しない

§10 の攻撃手順で、手順4(撃破判定)で撃破された式神は手順5(反撃)を行わない。

### 裁定3: 霊力獲得は「盤面から墓地へ行くとき」全般

§9.5 の霊力獲得は、戦闘撃破に限らず**カード効果による破壊・生贄を含む**、盤面→墓地の移動すべてで発生する(所有者が `DESTROY_MANA_GAIN` を獲得)。例外は個別カードのテキストに記載される(例:「この効果による破壊では霊力を獲得しない」)。
なお生命価による生命喪失は「撃破」時のみで、こちらは§9.5どおり。

### 裁定4: 霊力の保有上限

constants に `MAX_MANA = 15` を追加(§17)。超過分は消滅する。

## 3. 実装上の解釈メモ(シミュレーションで確定済みの挙動)

RULE_SPECの記述だけでは迷いうる点の、シミュ側で採用した解釈。エンジンも同じ解釈にすること:

| # | 項目 | 解釈 |
|---|---|---|
| 1 | 霊力補充のタイミング | §6のとおり**自ターン終了時**(開始時ではない)。つまり第1ターンは初期霊力のみで行動する |
| 2 | 召喚の位置制限 | 自分の式神が0体なら任意の空きマス。1体以上いるなら「いずれかの式神(敵味方問わず)と上下左右で隣接する空きマス」のみ(§9.2) |
| 3 | 召喚とメインフェイズ | 召喚(と召喚攻撃の解決)を行うとメインフェイズ終了。**霊具・再命令は召喚の前にしか行えない**(§6.2) |
| 4 | 属性HP修正 | マスにいる間、**現在HPに対して**適用・解除(最大HPではない)。効果で移動した場合は旧マスの修正を外し新マスの修正を付け、その時点でHP≤0なら撃破判定(§11.2の例示どおり) |
| 5 | MAX_HP | 属性修正・回復・強化を含むあらゆる手段でHPは `MAX_HP` を超えない |
| 6 | 弱点・反撃 | 弱点ボーナスと反撃は**物理攻撃のみ**。術式(魔道)攻撃は弱点ボーナスなし・反撃を受けない |
| 7 | デッキ切れ | 墓地をシャッフルして新しいデッキに(§15)。デッキ切れ敗北なし。墓地も空なら単に引けない |
| 8 | 手札補充 | ターン終了時に5枚まで引く/超過分は選んで捨てる。ターン中のドローで一時的に5枚を超えるのは可 |
| 9 | 制圧勝利の判定 | 自ターン終了時のみ(§6.3)。相手ターン中に5マスになっても勝利しない |
| 10 | 秘術 | 詳細未定(§13)。実装はスタブでよい。設計方向は `docs/hijutsu-design-draft.md` |

## 4. 素体テストカードセット(動作確認用)

実カードプールは改訂中のため、まずこの素体セットでエンジンを動かすことを推奨。バランス検証済みの構成なので、**そのまま最初のプレイテストシナリオになる**。

### 式神(6種)

全種共通: 攻撃範囲=前方1マス、対象1体、反撃範囲=前方1マス、弱点=後方1マス、物理、効果なし。

```json
[
  { "id": "vanilla_c2", "name": "素体・二", "cost": 2, "hp": 2, "atk": 1, "reactivation_cost": 2, "life_value": 1 },
  { "id": "vanilla_c3", "name": "素体・三", "cost": 3, "hp": 3, "atk": 2, "reactivation_cost": 3, "life_value": 1 },
  { "id": "vanilla_c4", "name": "素体・四", "cost": 4, "hp": 4, "atk": 2, "reactivation_cost": 3, "life_value": 1 },
  { "id": "vanilla_c5", "name": "素体・五", "cost": 5, "hp": 6, "atk": 3, "reactivation_cost": 3, "life_value": 3 },
  { "id": "vanilla_c6", "name": "素体・六", "cost": 6, "hp": 7, "atk": 3, "reactivation_cost": 3, "life_value": 3 },
  { "id": "vanilla_c7", "name": "素体・七", "cost": 7, "hp": 8, "atk": 4, "reactivation_cost": 4, "life_value": 4 }
]
```

(再命令コストは2026-07-07にheavyカーブ 2/3/3/3/3/4 へ改定済み。回転攻撃ルールとセットで採用)

属性はカード定義ではなく**デッキ生成時に 地/水/火/風 から均等ランダムで割り当てる**(空は式神に割り当てない)。

### 霊具(4種・数値は仮)

```json
[
  { "id": "item_removal", "name": "素体・除去", "cost": 2, "effect": "盤面の敵式神1体に2ダメージ(全域から対象選択、反撃なし)" },
  { "id": "item_economy", "name": "素体・経済", "cost": 1, "effect": "霊力を2得る" },
  { "id": "item_buff",    "name": "素体・強化", "cost": 2, "effect": "味方式神1体のHP+2(永続、MAX_HPまで)" },
  { "id": "item_draw",    "name": "素体・ドロー", "cost": 1, "effect": "カードを1枚引く" }
]
```

**注意: 霊具の効果・数値は仮**(バランス検証用のプレースホルダー)。本設計はフェーズ4で行われるため、霊具もデータ駆動で差し替えられる作りを推奨。

```
水 火 風
地 空 水
火 風 地
```

### 動作確認の目安(シミュとの突き合わせ)

このデッキのミラー対戦をAIなし(人間2人)または簡易AIで回したとき、シミュ上の期待値は: **平均約8〜9ラウンド、制圧勝ちがほぼ100%、先手勝率約50%**(2026-07-07採用構成: 再命令heavy+回転攻撃ルールで平均8.19R・先手50.0%)。ゲームが5ラウンドで終わり続ける/15ラウンド超が頻発する場合は、ルール実装のどこかが仕様とズレている可能性が高い(疑うべき筆頭: 召喚時の即攻撃、召喚の隣接制限、霊力補充のタイミング、回転攻撃の枠消費)。
