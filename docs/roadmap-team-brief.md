## 進め方の考え方: 「下から積んで、1段ずつ確定する」

カードゲームのバランスは「基礎数値 × カードの形状 × 効果 × 秘術」の掛け算で決まります。全部を同時にいじると、何が原因で壊れたのか分からなくなる。なので私たちは**1段ずつシミュレーションで確定させて、次の段に進む**方式を取っています。

```
【済】 基礎数値      … コスト別のHP/ATK/再命令コスト、マナカーブ、ゲーム定数
【済】 霊具の枠      … デッキ24枚=式神16+霊具8。仮の霊具でゲームが目標の長さになることを確認
【今】 形状の値段    … 攻撃範囲が広い・反撃されない等が「コスト何個分」かを実測
【次】 効果と実霊具  … カード効果を1種類ずつ値付け
【次】 秘術         … 生命を支払う大技(劣勢時の保険、の方向で設計中)
【次】 実カード反映  … 既存104枚を新数値・新属性で作り直し、陣営デッキを構築
【次】 プレイテスト  … 人間の体感で最終校正
```

各段の確定時には、シード・再現コマンド付きのスナップショットを凍結し、次の段はそれを対照群にします。つまり**後の段で数字が壊れたら、どの段で壊れたか必ず特定できる**仕組みです。

## いまの到達点(2026-07-05時点)

- 約10万試合のシミュレーションで基礎数値を確定(ゲームは平均約9ラウンド、陣取り決着ほぼ100%、先手勝率49〜51%、王手の返し率約55%)
- 「王手を返せるか」はAIの腕ではなくカード数値の性質として計測できる体制(総当たり検証つき)
- エンジン実装用の引き継ぎパック([engine-handoff-pack.md](engine-handoff-pack.md))を発行済み

## FAQ: 「再命令(再行動)のコストが高すぎるのでは?」

もっともな指摘なので、検証の経緯と現在の判断を共有します。

**Q1. 実際、使われていないのでは?**
はい、少ないです。攻撃の再命令は1試合あたり約2.4回で、目標帯(2.5〜4.5)の下端。この認識は共有しています。

**Q2. コストを下げれば解決?**
これは**シミュレーションで検証済みで、答えはノー**でした。

- 「霊力が足りていて、攻撃できる相手もいるのに、再命令しなかったターン」を直接計測したところ約70%。しかも**コストを最安(半額以下)にしても約69%でほぼ変わらない**。つまり払えないから使わないのではなく、「使う価値のある場面が少ない」が原因
- コストを下げた変種は3段階試したが、使用量はほぼ増えず、代わりに先手後手バランスが崩れ(先手勝率39〜45%)、撃破数だけが増えてゲームの健全性が下がった

**Q3. ではなぜ「使う価値のある場面」が少ない?**
現在のシミュレーションは検証のために「全カードが前方1マスしか攻撃できない素体」で動いているからです。攻撃範囲が狭く、攻撃を報酬する効果もまだ入っていない——**攻撃の価値が構造的に一番低い状態**で測っています。

**Q4. じゃあいつ直すの?**
コストの最終調整は「コスト vs 価値」の天秤なので、**価値側が本物になってから**が正しい順番です。次のフェーズ(形状=攻撃範囲の導入)と、その次(「攻撃するたび〜」系の効果)で攻撃の価値が実装されます。各フェーズ完了時に再命令の使用回数を再計測し、**それでも目標帯(2.5〜4.5回)に入らなければ、コスト削減を正式にテーブルに載せます**。そのとき何が壊れやすいか(先手バランス・撃破インフレ)のデータも既に手元にあります。

**要約**: 「高すぎるかも」は正当な仮説だが、計測上の犯人はコストではなく攻撃の価値。価値のレイヤーが入る前にコストをいじるのは、2つの変数を同時に動かして原因を分からなくする悪手なので、**順番として後**に置いている。放置ではなく、監視KPIと再開条件つき。

## 意見の挟みどころ

- **数値・KPIへの異論**: いつでも歓迎。ただし上のFAQのように「シミュで検証→データで回答」の形にするので、疑問は具体的な形(「◯◯が高すぎる/少なすぎるのでは」)でもらえると検証に落としやすい
- **効果・カードのアイデア**: フェーズ4(効果の値付け)前が出しどき。値付けの仕組みに乗せて公平に評価できる
- **秘術・ドラフト方式**: 設計たたき台([hijutsu-design-draft.md](hijutsu-design-draft.md) / [faction-draft-format-draft.md](faction-draft-format-draft.md))へのフィードバック募集中
- **体感の違和感**: エンジンが遊べる状態になったら、コアループ検証プレイテストを最初に実施予定。「数字は良くても体感が悪い」の指摘はそこが本番
