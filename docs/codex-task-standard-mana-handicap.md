# Codex Task: standard採用パスの追試(初期霊力ハンデ掃引)

Status: 実行待ち
Date: 2026-07-07

前提資料: `docs/codex-task-phase2d-rotate-attack.md` と `docs/baselines/phase2d-rotate-attack.md`(フェーズ2d)

## 目的

standardコスト(2/2/2/3/3/3)の先手勝率46.9%(後手有利)を、**dmではなく初期霊力ハンデで**帯内(50〜53%)に戻せるかを検証する。dmはルール定数(=1)として固定し、バランス調整は純粋な調整弁である初期霊力側で行う方針。

## 実行(4ラン×2000試合)

すべて seed `20260702`(フェーズ2c/2dと同一シード系列)、oracle あり、depth 6、
再命令コスト standard(2/2/2/3/3/3)、**DESTROY_MANA_GAIN=1 固定**、他はv3構成固定:

| ラン | 初期霊力(先手/後手) | ②ルール | 目的 |
|---|---|---|---|
| S1 | 3 / 3(後手+0) | なし | ハンデ撤廃でどこまで先手に戻るか |
| S2 | 3 / 3(後手+0) | あり | 同+②ルール併用 |
| S3 | 3 / 4(後手+1、現行) | あり | 対照補完: standard+②+現行ハンデ(2dで未実施の組) |
| S4 | 3 / 5(後手+2) | なし | ハンデ感度の下側確認(用量反応の3点目) |

計測はフェーズ2dと同一(全KPI+勝敗別使用+返し手勝率+計測B/C/D)。

**追加計測(全ラン)**: 「4チェックの返答に1回以上**失敗した**プレイヤーのその試合の勝率」。返し手勝率(成功側)との対比で「返す価値」を直接示すため。可能なら「成功も失敗も両方あるプレイヤー」は両方に計上し、件数も併記。
