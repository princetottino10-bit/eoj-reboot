## やること1: 印刷キットのデータ照合

`scripts/generate-print-kit-0718.mjs`(と生成物 `print-kit-0718-clean.html`)のカードデータを `docs/cardlist-0718.md` と全項目突き合わせ、不一致を報告する。

照合項目(9種すべて): 名前 / コスト / HP / ATK / 再命令コスト / 生命価 / 攻撃範囲=反撃範囲の座標 / 弱点座標 / 枚数 / 属性割当(各プレイヤー地水火風×4、表示は「地」)。

特に注意して見る箇所:
- 生命価の段差(C2-4=1、C5-6=3、C7=4)
- 龍だけ再命令コスト4(他のC3以上は3)
- 両面: 後1([[-1,0]])が**攻撃範囲に入っている**(弱点は左右)
- 九尾: [[1,1],[1,-1],[1,0],[0,1],[0,-1]]の5マス、弱点[[-1,0]]
- カード下端のテキストが描画枠から見切れていないか(HTMLのCSS上の問題として)

不一致があれば**スクリプト側を修正して再生成**(cardlist-0718.mdが正)。cardlist側が誤りと思われる場合は修正せず報告のみ。

## やること2: 印刷仕様そのままの確認ラン(shapes-0718-p)

印刷キットの仕様はT2シミュと2点ズレている:
1. **C4のHPが5**(T2ではC4=4。印刷版はC2/C3/C4に+1)
2. **九尾(C6)が5マス形状**(T2計測時は全周4方向)

印刷仕様と完全一致の構成で1本回し、実測期待値を取る:

- デッキ: 鬼火4 / 小鬼4 / 槍霊1 / 薙刀霊1 / 鬼2 / 両面1 / 仁王1 / 九尾1 / 龍1(=Mデッキと同一構成、名前は新名)
- ステータス: **HP = C2:3 / C3:4 / C4:5 / C5:6 / C6:7 / C7:8**(印刷版どおり)。ATK・再命令・生命価はv3.1どおり
- 九尾: attack_cells=counter_cells=[[1,1],[1,-1],[1,0],[0,1],[0,-1]]、weakness=[[-1,0]]。**斜め座標の攻撃・反撃・回転処理が正しく動くことを最初に小規模で確認**してから本ランへ(エンジン初の斜め使用の可能性)
- 反撃範囲=攻撃範囲、他ルール裁定は従来どおり
- ミラー2000試合、seed `20260702`、oracleあり、depth 6
- 報告: 全KPIを「T2(旧計測) / 印刷仕様(今回)」の2列比較+形状別内訳(特に九尾の撃破/被撃破/②③)

参考目標(判定は人間側): 平均9〜10.5R、生命勝ち≤10%、占拠勝ち≥90%。C4硬化で平均Rがやや伸びる方向の見込み。
