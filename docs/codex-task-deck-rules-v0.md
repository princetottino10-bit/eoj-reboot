## 実装(scripts/explore-stat-curves.cjs)

1. `buildDeck` を変更:
   - デッキ = クリーチャー16枚(派閥全10種を1枚ずつ + 追加6枚をランダムに選択、同名は合計2枚まで)
   - 重複選択はランダムのまま(デッキ構築AIの賢さを計測に混ぜない)
2. ファティーグを実装:
   - ターン終了時の手札補充でデッキが空だった場合、そのプレイヤーはライフを1失う(1ターンに1回まで)
   - ファティーグ由来のライフ敗北は通常のライフ敗北と区別できるようにフラグを分けて集計する
3. 計測を追加:
   - デッキ切れ発生ラウンドの分布(初めてデッキが空になったラウンド)
   - ファティーグ由来の敗北率
   - 既存KPIは全て維持
4. `scoreSummary` を v1.1 のKPI帯に更新(まだの場合):
   - 平均R: 8〜12中心 / 4チェック返し: 45〜55%(上限超過も減点) / 回転: 5〜9 / 占拠: 85%以上 / ライフ: 8%以下 / 先手: 50〜53% / P90: 23R超で減点
