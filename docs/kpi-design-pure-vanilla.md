# KPI設計書: 純粋バニラ探索

Status: **帯確定済み(フェーズ1完了、2026-07-05)** — 確定帯は文末の「確定KPI帯」を参照。本文のKPI体系の説明は設計経緯として残す
Date: 2026-07-05

## 目的と前提

これまでのシミュレーション(バニラ基準v1、回転感度測定)は実カードの形状とプール構成を含んだ計測であり、純粋な数値設計の基盤としては不適切だった。本探索では:

- **カードリストを無視**した純粋素体でシミュレーションする
- **ステータスカーブ**(HP/ATK/再行動)と**デッキのコスト分布**(どのコストを何枚入れるか)を両方設計する
- ドラフトは**陣営デッキ単位**(デッキは設計側が構築し、プレイヤーはカードを選ばない)。コスト分布は「設計者が定めるスタンダード」として1つ設計する。ただし各陣営デッキはカーブに個性を持つため、**答えは点ではなく中心+許容半径**(=陣営デッキのカーブ個性を振ってよい幅)で出す

## 純粋素体の仕様

全クリーチャーを以下のテンプレートに統一(コストだけが変数):

| 項目 | 値 |
|---|---|
| 攻撃範囲 | 前方1マス(1,0)のみ |
| 攻撃対象 | 1体 |
| 反撃範囲 | 前方1マス(1,0)のみ |
| 弱点範囲 | 後方1マス(-1,0)のみ |
| 攻撃タイプ | 物理のみ(魔道なし) |
| 属性 | 火/水/土/木を均等にランダム割当(属性マスルールは維持) |
| キーワード・効果 | なし |
| HP/ATK/再行動 | コスト別カーブから付与 |

形状税は適用対象なし(全カード同形状のため)。回転コストは攻撃再行動コストと同額(R1未採用のため)。

## デッキ・ルール前提

- デッキ: クリーチャー16枚(deck-rules-draft.md準拠。アイテム8枠は効果レイヤーまで空欄)
- ファティーグ: あり(補充時デッキ空でライフ1/ターン)
- その他ルール: v1.1ドキュメントの確定ルール表に従う(マナ+3/上限15、後手マナ+1、ライフ15、撃破1/3/4、弱点+2、属性±2、召喚時即攻撃)

## KPI体系

### A. 体験KPI(帯あり・選定の主目的関数)

体験目標(8〜12R、陣取り主役、王手応酬)から導出。プールに依存しないため旧帯を継続:

| KPI | 目標帯 |
|---|---|
| 平均ラウンド | 8〜12R |
| P90ラウンド | 20R以下(暫定) |
| 先手勝率 | 50〜53% |
| 占拠勝ち率 | 85%以上 |
| ライフ勝ち率 | 8%以下(ファティーグ由来を含む。内訳は分離集計) |
| 4チェック返し率 | 45〜55% |
| 撃破数/試合 | 4〜8 |

### B. 行動量KPI(帯はキャリブレーション後に確定)

「結果は良いが行動として使われていない」の検出用。旧環境の値は形状・デッキが違うため帯として持ち込まない:

| KPI | 観測内容 |
|---|---|
| **再行動回数/試合** | 攻撃再行動と回転を分離して集計 |
| 回転シェア | 再行動中の回転比率 |
| 手動攻撃 vs 召喚時攻撃の比率 | 攻撃判断が召喚のオマケに吸収されていないか |
| 弱点撃破率 | 位置取りゲームの成立度(純粋素体では全カード後方弱点のため値域が変わる) |
| 再行動可能未使用率 | マナも対象もあるのに再行動しなかったターン率(コスト過剰の診断) |

### C. カーブ健全性KPI(コスト分布の直接評価・新設)

| KPI | 観測内容 |
|---|---|
| 手詰まりターン率 | 召喚可能なカードが手札とマナの噛み合いで存在しないターンの割合 |
| マナ利用率 | 支出/収入。低=分布が軽すぎ、高+手詰まり併発=重すぎ |
| 初召喚〜盤面3体到達ラウンド | 序盤テンポが3〜4マス攻防の体験に間に合うか |
| デッキ切れ到達ラウンド分布・ファティーグ敗北率 | 分布×デッキ枚数の整合 |
| マナ溢れ/試合 | 上限15との整合 |

### D. 頑健性KPI(ドラフト前提・新設)

スタンダード分布の候補に対して:

| KPI | 目標(暫定) |
|---|---|
| 近傍分布のKPI維持 | 標準から±2枚シフトした分布でも体験KPI帯内 |
| ズレ対戦の勝率偏り | 軽寄り vs 重寄り等の対戦で勝率55〜60%以下(陣営デッキ間のカーブ個性差が素の勝率差になりすぎない) |

### E. 計測器KPI(健全性チェック)

- オラクル実現率 95%以上(チェック探索がオラクル上限に届いているか)
- タイムアウト率 3%以下

## 探索プロセス

### 段階1: キャリブレーション(スコア関数なし)

- カーブ: 現行勝者(standard/high/heavy)+ lean系1種 + react standard系/light系の計4種(再行動コストはheavy/standard/lightの3点比較で感度を分離して読めるようにする)
- コスト分布(16枚をC2〜C7に配分)5候補:

| 分布 | C2 | C3 | C4 | C5 | C6 | C7 |
|---|---:|---:|---:|---:|---:|---:|
| 軽速 | 5 | 5 | 3 | 2 | 1 | 0 |
| 低中 | 4 | 4 | 4 | 2 | 1 | 1 |
| バランス | 3 | 4 | 3 | 3 | 2 | 1 |
| 中重 | 2 | 3 | 3 | 3 | 3 | 2 |
| 重量 | 2 | 2 | 3 | 3 | 3 | 3 |

- ミラー対戦(同分布同士)で4カーブ×5分布、各1000試合
- 全KPIを帯判定なしで記録し、値域を観測する

### 段階2: 帯とスコア関数の確定

- キャリブレーション結果を見て、B/C区分のKPI帯を合意の上で確定
- `scoreSummary` を新KPI体系に書き換える(旧目標のハードコード事故の再発防止)

### 段階3: 本探索

- カーブ×分布のグリッドをスコア関数で選定
- 上位候補について: 近傍±2枚シフトの頑健性テスト、ズレ分布同士のクロス対戦行列
- 答えは「スタンダード分布(中心)+許容半径」の形で確定する

### 段階4以降(本探索の後)

- 形状の値付け: 素体との差分実験(範囲+1、対象+1、全域魔道等を1変数ずつ足して勝率影響を測る)→ 形状税の経験的再導出
- 実カードプールへの適用とアイテム/効果レイヤー

(注: 回転コストの分離検証は当面実施しない。回転=攻撃コスト同額を前提とする)

- キャリブレーションv2で「再行動非使用はコストではなく価値の問題」と確定(全変種で可能未使用率68〜75%、lightカーブでも改善せず)。再行動コストの追加調整は行わず、価値側(形状・効果)のレイヤーで扱う
- 引き分け(トラッカー#3): 純粋バニラでは全変種0%。効果レイヤー後に再評価
- DECK_SIZE=24(トラッカー#5): デッキ切れがほぼ発生しないことを確認。暫定継続

## フェーズ2b計測後の定義見直し(承認済み 2026-07-05)

フェーズ2b(素体霊具)の結果、B1は帯外が3項目出たが、うち2つは**数値の問題ではなく指標の定義が霊具の存在を想定していなかった**ことによる見かけの違反である。以下の定義変更を提案する:

1. **弱点撃破率**(B1: 37.2%で帯≥45%に違反): 現定義は「全キル中の弱点経由率」で、霊具キル(位置取り無関係)が分母に入り希釈される。**「攻撃によるキル中の弱点率」に定義変更**を提案。換算するとB1は約45%で帯の縁に収まり、位置取りゲーム性は劣化していない
2. **手詰まりターン率**(B1: 23.4%で帯≤20%に違反): 現定義は「式神を召喚できないターン」で、手札が霊具に置き換わった分が機械的に増える。しかし霊具を使えるターンは体験として手詰まりではない。**「式神も霊具も何も使えないターン」に定義変更**を提案
3. **攻撃再行動2.4**(帯2.5〜4.5に−0.1): 霊具導入で増える見込みだったが、霊具がマナを食い合い増えなかった。形状レイヤー(攻撃範囲拡大=殴る価値の増加)での回復を次の観察点とする。定義変更はしない

また、除去霊具の枚数が「ゲーム長」(7.8→8.9→9.8R)と「王手返し率」(43.8→55.2→63.4%)の**両方を動かす最強のチューニングノブ**であることが確定した。B1(除去2枚)の返し率55.2%は帯上限ちょうどであり、フェーズ4の実霊具設計では除去系の枚数・強度をこの用量反応を基準に配分する。

計測上の注意: 現実装ではオラクルの行動空間に除去霊具のみを含むため、実返し率がオラクル比100%を僅かに超える(B1: 101.3%)。経済・強化・ドローが返答に寄与した分で、計測器の異常ではない。
