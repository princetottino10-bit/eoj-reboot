# Codex Task: フェーズ3 形状の値付け(実測導出)

Status: 実行待ち
前提資料(先に読むこと):

- `docs/baselines/phase2b-item-layer.md` — 基準環境(B1: 式神16+素体霊具8)
- `docs/kpi-design-pure-vanilla.md` — 確定KPI帯と定義(2b後改訂込み)
- `docs/card-audit-draft.md` §4 — 防御形状の測定が必要な理由(実プールからの申し送り)

目的: 攻撃範囲・対象数・術式・反撃/弱点形状の「値段」をシミュレーションで実測し、**経験則だった形状税を実測ベースの換算表に置き換える**。

## ルール上の注意(毎回明示)

- 回転は式神1体につき1ターン1回まで(RULE_SPECの記載に関わらず裁定済みの例外)
- 撃破判定で撃破された式神は反撃しない
- 盤面から墓地へ送られた式神の所有者は `DESTROY_MANA_GAIN=1` を獲得
- 環境は**B1構成**(v3カーブ+lowmid分布+素体霊具8枚)を基準とする

## 方法論: 勝率デルタのコスト換算

### ステップ1: 為替レートの校正

「勝率何ポイント = コスト1相当」の換算レートを先に測る。

- **CAL1**: 中心デッキの C3スロット4枚 を「C4のステータス(4/2/3, 生命価1)を持つがコスト3のまま」の式神に置き換えたデッキ vs 中心デッキ。2000試合
- **CAL2**: 同様に「C3ステータスのままHPだけ+1」版。2000試合
- **CAL3**: 同様に「ATKだけ+1」版。2000試合

CAL1の勝率デルタが「コスト1の価値」の基準レート。CAL2/3はステータス単体の内訳(HPとATKの交換比)の参考値。

### ステップ2: 形状変種の測定

C3スロット4枚を「C3ステータスのまま、形状だけ変えた」式神に置き換え、中心デッキと対戦(**片側置換**)。各2000試合。勝率デルタ ÷ 換算レート = その形状のコスト価値。

攻撃形状(基準: 前方1マス・1対象・物理):

| ID | 形状 |
|---|---|
| S1 | 前方2マス・選択1体(射程+1) |
| S2 | 前方2マス・貫通(2体同時) |
| S3 | 前+左右3マス・選択1体 |
| S4 | 前+左右3マス・同時 |
| S5 | 前後2マス・同時 |
| S6 | 全周4方向・選択1体 |
| S7 | 全周4方向・同時 |
| S8 | 全域術式パッケージ(全域選択・反撃を受けない・死角ボーナスなし。魔道扱い) |

防御形状(基準: 反撃=前方1、弱点=後方1):

| ID | 形状 |
|---|---|
| D1 | 反撃なし(counter無し、物理のまま) |
| D2 | 全周反撃 |
| D3 | 弱点なし |
| D4 | 弱点が側面2マス(後方は弱点でない) |

### ステップ3: コスト帯スケーリングの抜き取り確認

形状の価値はATKに比例する可能性が高い(範囲が広い=高ATKほど強い)。S3(前+左右選択)のみ、**C5スロット2枚への適用版**でも測定し、C3との価格差を確認する。差が大きければ「形状価格はコスト帯別」、小さければ「一律」で換算表を作る。

### ステップ4: 健全性ミラー(上位形状のみ)

価格の高かった上位3形状について、両者とも変種を積んだミラー2000試合を実施し、KPI(弱点撃破率・王手返し率・平均R)への副作用を記録する。特にD3(弱点なし)とS8(術式)は位置取りゲーム性(弱点撃破率)を直接損なう疑いがあるため必須。
